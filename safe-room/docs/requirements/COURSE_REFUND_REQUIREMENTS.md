---# 课程退课需求文档

> 版本：v1.0
> 更新日期：2025-11-15

## 0. 设计关键词（Design Keywords）

- 透明化 · 责任共担 · 高效处理
- 会员退课流程应清晰透明，责任明确划分，处理流程高效可靠

---

## 1. 页面整体逻辑结构

| 顺序 | 区块 | 目的 |
| --- | --- | --- |
| ① | 退课申请流程引导 | 告知用户退课流程，收集必要信息 |
| ② | 课程选择与信息确认 | 选择需退课的预约，展示课程详情 |
| ③ | 退课原因说明 | 详细说明退课原因，便于审核 |
| ④ | 审核状态跟踪 | 实时查看退课申请处理进度 |
| ⑤ | 退款处理反馈 | 审核通过后的退款流程说明 |

---

## 2. 视觉风格规范

### 2.1 颜色

| 名称 | 颜色 | 用途 |
| --- | --- | --- |
| 科技黄 | `#FDD835` | 主 CTA、进度指示 |
| 深黑背景 | `#0A0A0A` | 页面背景 |
| 科技灰 | `#1A1A1A` | 卡片/分区背景 |
| 冷灰 | `#9EA1A6` | 辅助文字 |
| 危险红 | `#F44336` | 退课相关警告 |
| 成功绿 | `#4CAF50` | 审核通过状态 |

### 2.2 动效风格

- 步骤指示器：GSAP 实现步骤间平滑过渡
- 状态更新：审核状态变更时触发动画反馈
- Hover 行为：上浮 4px，发光增强 8%
- 页面滚动：步骤区块依次淡入，营造流程感

### 2.3 卡片与组件规范

- 圆角：18px
- 背景：黑色玻璃拟物，`backdrop-filter: blur(8-12px)`
- 边框：金色微光 0.5~2px，根据审核状态变化颜色
- 阴影：`0 20px 40px rgba(0, 0, 0, 0.45)`

## 3. 功能模块说明

### 3.1 基础CRUD操作

| 功能 | 说明 | 数据来源 | 路由/接口 |
| --- | --- | --- | --- |
| 退课申请列表 | 展示所有退课申请记录 | `kechengtuike` 表分页查询 | `/kechengtuike/page` |
| 申请详情查看 | 查看单个退课申请详细信息 | `kechengtuike` 表 + 关联查询 | `/kechengtuike/detail/{id}` |
| 新增退课申请 | 创建新的退课申请 | 用户表单输入 | `/kechengtuike/add` |
| 编辑申请信息 | 修改退课申请内容 | 表单数据更新 | `/kechengtuike/update` |
| 删除申请记录 | 删除无效退课申请 | 前端确认后提交 | `/kechengtuike/delete` |

### 3.2 审核状态管理

| 功能 | 说明 | 数据来源 | 路由/接口 |
| --- | --- | --- | --- |
| 审核状态标识 | 显示申请的审核状态 | `kechengtuike.sfsh` 字段 | 前端状态映射 |
| 状态筛选查询 | 按审核状态筛选申请 | 后端条件查询 | `/kechengtuike/page?sfsh=...` |
| 批量审核操作 | 管理员批量审核申请 | 前端多选 + 批量更新 | `/kechengtuike/batch/update` |

### 3.3 退款处理流程

| 功能 | 说明 | 数据来源 | 路由/接口 |
| --- | --- | --- | --- |
| 退款状态跟踪 | 显示退款处理进度 | `kechengtuike.ispay` 字段 | 支付状态查询 |
| 退款金额展示 | 显示应退款金额 | 后端计算结果 | 订单关联数据 |
| 退款确认操作 | 确认退款已完成 | 管理员操作 | `/kechengtuike/confirmRefund` |

## 4. 交互与动效规范

| 维度 | 规范 |
| --- | --- |
| 动画时长 | 0.3s ~ 0.55s |
| 缓动 | `cubic-bezier(0.4, 0, 0.2, 1)` 或 `GSAP Power3.easeInOut` |
| Hover | 发光增强 2%~10%，浮起 2~5px |
| 滚动触发 | 步骤区块依次出现，间隔 0.2s |
| 空状态 | 无预约记录时显示鼓励续约插画 |
| 加载状态 | 步骤提交时显示粒子进度条 |
