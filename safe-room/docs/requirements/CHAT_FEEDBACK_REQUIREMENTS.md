## 0. 设计关键词（Design Keywords）

- 即时响应 · 贴心服务 · 问题解决
- 提供即时的客户服务通道，确保会员问题得到及时、专业、高效的解决

---

## 1. 页面整体逻辑结构

| 顺序 | 区块 | 目的 |
| --- | --- | --- |
| ① | 留言提交入口 | 提供便捷的留言提交界面 |
| ② | 留言列表展示 | 按时间倒序展示所有留言记录 |
| ③ | 回复状态管理 | 清晰区分待回复和已回复状态 |
| ④ | 客服信息展示 | 展示客服回复内容和回复时间 |
| ⑤ | 快捷操作入口 | 提供常用问题入口和联系方式 |

## 2. 视觉风格规范

### 2.1 颜色

| 名称 | 颜色 | 用途 |
| --- | --- | --- |
| 科技黄 | `#FDD835` | 主 CTA、提交按钮 |
| 深黑背景 | `#0A0A0A` | 页面背景 |
| 科技灰 | `#1A1A1A` | 卡片/分区背景 |
| 冷灰 | `#9EA1A6` | 辅助文字 |
| 等待蓝 | `#2196F3` | 待回复状态 |
| 成功绿 | `#4CAF50` | 已回复状态 |
| 消息橙 | `#FF9800` | 新消息提醒 |

### 2.2 动效风格

- 消息气泡：GSAP 实现消息发送时的弹出动画
- 状态变化：回复状态变更时的颜色渐变和图标动画
- 新消息提醒：Lottie 实现消息到达的呼吸动画
- Hover 行为：上浮 4px，发光增强 8%
- 页面滚动：消息列表滚动触发动画

### 2.3 卡片与组件规范

- 圆角：18px
- 背景：黑色玻璃拟物，`backdrop-filter: blur(8-12px)`
- 边框：根据消息状态变化颜色（蓝→绿渐变）
- 阴影：`0 20px 40px rgba(0, 0, 0, 0.45)`

## 3. 功能模块说明

### 3.1 留言列表展示

| 功能 | 说明 | 数据来源 | 路由/接口 |
| --- | --- | --- | --- |
| 留言列表展示 | 按时间倒序显示所有留言 | `chat` 表分页查询 | `/chat/autoSort` |
| 留言主题显示 | 显示留言主题和创建时间 | `chat.ask`、`chat.addtime` 字段 | 格式化显示 |
| 回复状态标识 | 区分待回复和已回复状态 | `chat.isreply` 字段 | 前端状态判断 |
| 回复内容展示 | 显示客服回复内容和回复时间 | `chat.reply`、`chat.replytime` 字段 | 条件渲染 |

### 3.2 留言创建入口

| 功能 | 说明 | 数据来源 | 路由/接口 |
| --- | --- | --- | --- |
| 创建留言按钮 | 跳转到留言创建页面 | 前端路由跳转 | `/index/chatAdd` |
| 新留言引导 | 页面顶部提供创建入口 | 静态UI元素 | 本地状态管理 |

### 3.3 基础状态管理

| 功能 | 说明 | 数据来源 | 路由/接口 |
| --- | --- | --- | --- |
| 加载状态显示 | 显示数据加载进度 | 前端loading状态 | Element Plus v-loading |
| 空状态处理 | 无留言时显示空状态提示 | 前端数据判断 | Element Plus Empty组件 |

---

## 4. 交互与动效规范

| 维度 | 规范 |
| --- | --- |
| 动画时长 | 0.3s ~ 0.55s |
| 缓动 | `cubic-bezier(0.4, 0, 0.2, 1)` 或 `GSAP Power3.easeInOut` |
| Hover | 发光增强 2%~10%，浮起 2~5px |
| 滚动触发 | 消息卡片依次淡入，间隔 0.1s |
| 空状态 | 无留言时显示客服欢迎插画 |
| 加载状态 | 消息加载时显示骨架屏 |

## 5. 响应式策略

| 终端 | 处理 |
| --- | --- |
| PC（>1200px） | 左右布局：留言列表/提交表单 |
| Pad（768-1200px） | 单列布局，提交表单抽屉化 |
| Mobile（<768px） | 卡片栈式，消息气泡化显示 |
