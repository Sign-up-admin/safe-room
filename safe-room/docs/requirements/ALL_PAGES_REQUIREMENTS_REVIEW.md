## 1. 概述

- 目的：在首页需求文档 (`docs/HOMEPAGE_REQUIREMENTS.md`) 的标准下，审视所有前台页面的功能需求、现状、工程实践，并识别不足。
- 范围：涉及前台 `src/pages`、`src/components/home`、`src/config/modules.ts` 等目录的全部功能模块。
- 方法：基于现有组件与模块配置，结合工程最佳实践（设计系统、交互一致性、性能、可访问性）进行对比分析。

## 4. 设计与交互一致性评估

### 4.1 视觉一致性

| 页面群组 | 一致性等级 | 主要问题 |
| --- | --- | --- |
| 首页 | ⭐⭐⭐⭐ | 基线标准 |
| 首页组件 (`src/components/home`) | ⭐⭐⭐ | 维持统一视觉，但缺文档化 |
| 模块页 (ModuleListPage 系列) | ⭐ | 继承 Element Plus 默认皮肤；未引入黑黄主题 |
| 自定义页（个人中心 / 新闻） | ⭐⭐ | 局部定制但未匹配首页科技感 |

### 4.2 动效与交互

- Hover 动效：多数页面使用默认 hover，无统一发光/上浮效果。
- 页面过渡：仅首页引入 GSAP 场景动画，其他页面无过渡。
- 加载反馈：依赖 Element Plus `v-loading`，缺少品牌化骨架屏或粒子加载。

### 4.3 响应式体验

- 首页：已对 PC/Pad/Mobile 制定策略。
- 其他页面：多为列表布局，缺少移动端断点优化（例如筛选面板与分页）。
- 图片资源：未统一 srcset/懒加载策略。

## 6. 主要不足与讨论

1. **视觉系统缺失**  
   - 问题：Module 级页面延续 Element Plus 默认皮肤，未继承首页黑黄科技感。  
   - 影响：品牌一致性差、体验割裂。  
   - 建议：建立 Design Token（颜色、字体、阴影、圆角）并通过 CSS 变量或 Element Plus 主题覆盖推广。

2. **动效与交互规范未通用**  
   - 问题：统一的 hover、过渡、滚动动效仅限首页。  
   - 建议：封装 `useHoverGlow`, `usePageTransition` 等组合式 API，与 `useAnimations.ts` 协同，并在模块列表、卡片上应用。

3. **业务功能深度不足**  
   - 示例：课程预约缺少冲突检测；教练详情缺少预约 CTA；支付页缺少流程指示；讨论区缺少互动。  
   - 建议：为关键业务补充交互流程（步骤条、状态提示、通知联动），提升可用性。

4. **响应式体验不均衡**  
   - 问题：多数列表页在 <768px 时排版拥挤。  
   - 建议：为 Module 组件增加断点布局、卡片折叠、筛选抽屉。

5. **文档与规范缺失**  
   - 目前仅首页有需求文档；组件、模块、设计系统缺描述。  
   - 建议：为主要页面补写需求说明，建立组件文档（Storybook/Markdown），并形成设计系统手册。

6. **可访问性与系统反馈不足**  
   - 缺少 ARIA 标签、键盘导航、骨架屏、空状态。  
   - 建议：在公共组件层补齐可访问性属性；设计统一空状态与骨架屏。

## 7. 改进路线与优先级

### P0（短期，1-2 个迭代）
1. 设计系统：提炼首页视觉元素，输出 Design Token，与 Element Plus 主题联动。
2. 核心业务增强：课程/私教预约增加冲突检测、流程反馈；支付流程可视化。
3. 组件扩散：将首页组件库（SmartHeader、HeroSection、CoachNetwork 等）的视觉/交互规范推广到模块列表与详情。

### P1（中期，3-4 个迭代）
1. 动效库：封装动画/交互组合式 API，统一 hover/过渡/加载体验。
2. 响应式策略：为 Module 组件增加断点样式，提供移动端筛选抽屉。
3. 文档化：为主要页面编写需求文档，补充组件/设计系统说明。

### P2（长期，持续演进）
1. 可访问性：ARIA、键盘导航、屏幕阅读器支持。
2. 用户体验：骨架屏、空状态库、错误边界与上报机制。
3. 数据可视化：个人中心展示训练进度、会员生命周期。

---

## 8. 结论

- **已有优势**：模块化 CRUD 架构、完善的类型系统、首页视觉方案成熟、现代化栈已落地。
- **主要挑战**：品牌视觉与动效难以在全站复用，业务流程深度不足，文档与规范缺失。
- **建议行动**：以首页为基线建立设计与交互系统，并在高价值页面（课程/教练/预约/支付）优先落地；同步推进文档与可访问性建设，实现体验与工程的闭环。

> 通过以上改进，可在保持工程效率的同时，构建一致、沉浸、可信赖的健身房数字体验。
