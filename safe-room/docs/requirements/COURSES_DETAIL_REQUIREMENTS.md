---# 健身课程详情页需求文档（Course Detail v1.0）

> 版本：v1.0  
> 更新日期：2025-11-15  
> 适用路由：`/index/jianshenkechengDetail?id={courseId}`  
> 关联模块：`moduleKey = 'jianshenkecheng'`

---

## 0. 设计关键词

沉浸视觉 · 数据分解 · 训练路线 · 高光 CTA · 口碑佐证

---

## 1. 页面整体逻辑结构

| 顺序 | 区块 | 目的 |
| --- | --- | --- |
| ① | 课程 Hero | 展示课程封面视频/图、核心卖点、立即预约 CTA |
| ② | 训练收益矩阵 | 用图表呈现肌群/燃脂/体态提升指标 |
| ③ | 课程信息卡 | 详述课程内容、课程周期、适用人群、设备要求 |
| ④ | 预约流程 & 档期 | 选择时间段，快速预约，显示剩余名额 |
| ⑤ | 教练团队 | 列出授课教练，跳转教练详情 |
| ⑥ | 真实反馈 | 学员评价、完成度图表、社交证据 |
| ⑦ | 相关推荐 | 推荐同类型或下一阶段课程 |

---

## 2. 视觉风格规范

- 背景：深黑渐层 + 粒子流动
- Hero 区使用全宽视频蒙版 + 金色描边标题
- 数据区域使用雷达图/柱状图，颜色采用科技黄 + 冷灰
- 信息卡采用玻璃拟物 + 软阴影

---

## 3. 功能模块说明

### 3.1 课程 Hero

| 元素 | 说明 |
| --- | --- |
| 封面媒体 | 优先展示课程介绍视频（静音循环），无视频则用高清图并添加粒子动效 |
| 文案 | 主标题 + 副标题 + 3 个要点标签 |
| CTA | `立即预约`（跳转 `/index/kechengyuyue`，带 courseId），`咨询顾问` |
| 数据 | 热度值（点击数）、平均评分 |

### 3.2 训练收益矩阵

- 使用图表（ECharts/Three.js 线框）展示四个维度：力量、体能、柔韧、体态
- 支持切换“课程前后对比”“行业平均”

### 3.3 课程信息卡

| 信息点 | 描述 | 来源 |
| --- | --- | --- |
| 课程周期 | 课时数 + 周期 | `/jianshenkecheng/detail` |
| 适用人群 | 标签化展示 | 数据字段或配置 |
| 所需设备 | 列表 + 图标 | 课程字段/器材映射 |
| 注意事项 | 富文本/Markdown 支持 |

### 3.4 预约流程 & 档期

- 展示本周/本月可预约时间段（调用 `/kechengyuyue/list`）
- 时间选择器支持日历、时间段标签
- 冲突检测：若用户已预约同时间段课程，则提示更换
- 展示剩余名额、预约成功率

### 3.5 教练团队

- 默认展示授课主教练 + 助教
- 卡片包含头像、擅长领域、跳转按钮（`/index/jianshenjiaolianDetail`）

### 3.6 真实反馈

- 学员评价滑动组件，支持星级、标签、图片
- 数据统计：完成度、复购率、平均评分

### 3.7 相关推荐

- 根据课程类型/训练目标推荐 3~4 个课程
- 支持“下一阶段训练”引导

---

## 4. 交互与动效

| 交互 | 说明 |
| --- | --- |
| Hero 垂直滚动视差 | 背景视频慢速滚动，标题固定 |
| 图表动画 | 数据加载时逐步填充，切换场景时动画过渡 |
| 预约时间 hover | 高亮时段，展示剩余名额 |
| 评价滚动 | 自动滚动 + hover 暂停 |

---

## 5. 响应式策略

| 终端 | 处理 |
| --- | --- |
| PC | 双栏布局（媒体 + 信息），图表完整展示 |
| Pad | Hero 缩放，图表改为单列，预约模块折叠 |
| Mobile | 媒体改为静态图，图表简化为条形图，预约模块采用步骤化浮层 |

---

## 6. 技术栈与依赖

| 领域 | 实现 |
| --- | --- |
| 媒体 | video.js 或原生 video + 背景 Canvas 粒子 |
| 图表 | ECharts / D3 雷达图 |
| 数据 | 课程详情接口 + 预约接口聚合 |
| 状态 | Pinia（暂存用户预约意向） |
| 动效 | GSAP + ScrollTrigger |

---

## 8. 验收标准

| 维度 | 标准 |
| --- | --- |
| 功能 | 课程数据完整展示，可预约、可跳转教练详情 |
| 动效 | Hero/图表/评价滚动均符合帧率要求 |
| 性能 | 首屏加载 ≤ 3s，媒体懒加载 |
| 响应式 | 三端无布局错乱 |
| 可访问性 | 预约控件可键盘交互，图表提供文本替代 |

---

## 9. 备注

- 与课程列表/预约页共享课程数据模型
- 若课程视频托管第三方，需考虑跨域/预加载策略
